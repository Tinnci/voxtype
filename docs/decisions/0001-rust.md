# ADR 0001: Use Rust for the desktop service

- Status: accepted
- Date: 2026-07-15

## Decision

Use Rust for the VoxType daemon, CLI, protocol implementation, and initial KDE
desktop adapters.

## Rationale

- Strong ownership and concurrency model for long-lived audio and network tasks.
- Good fit for bounded streaming pipelines and explicit cancellation.
- Native binary deployment without a language runtime.
- Mature async, D-Bus, WebSocket, audio, and Linux desktop ecosystem.
- Easier sharing of domain types across daemon and CLI than a mixed-language
  first release.

Go remains a reasonable alternative for a standalone network service, but the
audio callback, desktop IPC, and fine-grained lifecycle requirements favor Rust
for this project.

## Consequences

- Unsafe FFI must be isolated and justified if direct PipeWire or desktop APIs
  require it.
- Dependency and crate maintenance quality must be evaluated during Phase 0.
- UI technology remains open; the tray/settings layer does not have to dictate
  the language of the daemon.
